## 1. Executive Summary

### 1.1 프로젝트 비전

**"누구나 AI로 비디오 콘텐츠를 만들고, 모든 채널에 자동으로 배포하는 세상"**

XCaster는 비디오 콘텐츠 제작의 모든 과정(기획-제작-배포-분석)을 AI와 자동화로 혁신하여, 크리에이터와 마케터가 **콘텐츠 자체에만 집중**할 수 있도록 지원하는 SaaS 플랫폼입니다.

### 1.2 핵심 가치 제안 (Value Proposition)

#### 문제점 (Problem)
1. **제작 부담**: 영상 제작에 높은 시간과 비용 소요
2. **멀티 플랫폼 관리**: 각 채널마다 개별 업로드와 관리 필요
3. **최적화 어려움**: 플랫폼별 최적 발행 시간 파악 어려움
4. **기술 진입장벽**: 영상 편집 툴 학습에 시간 소요

#### 솔루션 (Solution)
1. **AI 자동 생성**: 키워드만 입력하면 완성된 숏폼 영상 생성
2. **멀티채널 동시 배포**: 하나의 클릭으로 모든 플랫폼에 발행
3. **스마트 스케줄링**: AI 기반 최적 시간대 자동 예약
4. **노코드 제작**: 복잡한 툴 없이 직관적인 UI로 제작

### 1.3 타겟 시장

#### Primary Target
- **개인 크리에이터** (유튜버, 인플루언서)
  - 콘텐츠 제작 속도 향상
  - 멀티 플랫폼 운영 자동화

- **마케팅 대행사**
  - 클라이언트별 효율적 콘텐츠 관리
  - 대량 콘텐츠 자동 생성 및 배포

#### Secondary Target
- **기업 마케팅팀**: 브랜드 콘텐츠 일관성 유지
- **이커머스**: 상품 홍보 영상 + 라이브 커머스

### 1.4 비즈니스 목표

**Short-term (6개월)**
- MVP 출시 및 베타 테스터 100명 확보
- 월 평균 500개 이상 숏폼 생성
- 3개 이상 채널 연동 (YouTube, Facebook, Naver TV)

**Mid-term (1년)**
- 유료 가입자 1,000명 달성
- 5개 이상 플랫폼 지원
- 라이브 스트리밍 기능 정식 출시

**Long-term (2년)**
- ARR(연간 반복 매출) $1M 달성
- 기업용 Enterprise 플랜 출시
- 글로벌 시장 진출 (영어, 일본어 지원)

## 2. User Personas

**배경**
- 나이: 28세, 유튜버/인스타그래머
- 구독자: YouTube 5만, Instagram 3만
- 통증점: 영상 편집에 하루 3-4시간 소요, 여러 플랫폼 관리 번거로움

**목표**
- 제작 시간 절반으로 줄이기
- Instagram, YouTube, TikTok 동시 운영

**XCaster 사용 시나리오**
1. 아침에 키워드 5개 입력 → AI가 스크립트 생성
2. 점심시간에 생성된 영상 확인 및 간단 수정
3. 저녁에 모든 플랫폼에 예약 발행
4. 결과: **제작 시간 70% 감소**

- 통합 대시보드에서 성과 모니터링

**XCaster 사용 시나리오**
1. 팀원들과 클라이언트별 프로젝트 생성
2. 템플릿 기반으로 빠르게 대량 콘텐츠 제작
3. 클라이언트별 최적 시간대 자동 발행
4. 통합 분석 리포트로 주간 회의
5. 결과: **운영 효율 50% 향상, 클라이언트 만족도 증가**

- 라이브 커머스 간편하게 진행

2. 라이브 스튜디오에서 제품 소개 방송
3. 방송 중 할인 쿠폰 자동 노출
4. YouTube, Facebook 동시 송출
5. 결과: **매출 20% 증가, 라이브 운영 시간 절반**

## 3. Core Features (핵심 기능)

### 3.1 숏폼 자동 생성

#### 기능 개요
키워드만 입력하면 AI가 완성된 숏폼 비디오를 자동으로 제작합니다.

#### 상세 스펙

**Step 1: 키워드 입력**
- 사용자가 주제 키워드 입력 (예: "여름 휴가 패션 추천")
- 선택 옵션: 톤앤매너, 비디오 길이, 스타일

**Step 2: AI 시나리오 생성**
- Claude API로 스크립트 자동 작성
- 구조: 인트로 → 본론 → 아웃트로
- 사용자가 스크립트 수정 가능

**Step 3: 이미지/영상 생성**
- PiAPI (Flux): 고품질 이미지 생성
- PiAPI (Kling): AI 영상 클립 생성
- 사용자 업로드 이미지/영상 사용 가능

**Step 4: 음성 합성**
- ElevenLabs: 자연스러운 음성 내레이션
- 다양한 목소리 선택 가능
- 속도, 톤 조절

**Step 5: 비디오 합성**
- Creatomate: 자동 편집 및 효과 적용
- 템플릿 기반 빠른 제작
- 최종 결과물 미리보기

**Step 6: 최종 확인 및 저장**
- 프로젝트로 저장
- 채널별 배포 준비

#### 사용자 플로우
```
[키워드 입력]
  → [AI 스크립트 생성]
  → [스크립트 확인/수정]
  → [이미지/영상 생성]
  → [음성 합성]
  → [비디오 합성]
  → [미리보기]
  → [저장/배포]
```

#### 예상 제작 시간
- **기존 방식**: 2-4시간
- **XCaster**: 10-15분 (AI 처리 시간 포함)

### 3.2 멀티채널 배포

#### 지원 플랫폼 (Phase 1)
1. **YouTube**
   - OAuth 2.0 연동
   - 제목, 설명, 태그 자동 입력
   - 썸네일 자동 생성

2. **Facebook**
   - 페이지/그룹 선택
   - 자동 포스팅

3. **Naver TV**
   - 채널 연동
   - 카테고리 자동 분류

#### 지원 플랫폼 (Phase 2)
4. Twitch
5. TikTok
6. Instagram Reels

#### 배포 설정
- **즉시 발행**: 클릭 즉시 모든 채널에 배포
- **예약 발행**: 날짜/시간 설정
- **채널별 맞춤**: 플랫폼별 포맷 최적화
- **일괄 관리**: 단일 인터페이스에서 모든 채널 제어

#### OAuth 연동 플로우
```
[채널 연결하기]
  → [플랫폼 선택]
  → [OAuth 인증]
  → [권한 승인]
  → [연동 완료]
  → [대시보드에 표시]
```

#### 캘린더 뷰
- 월간/주간/일간 뷰 제공
- 드래그 앤 드롭으로 일정 조정
- 채널별 색상 구분

- 플랫폼별 최적 시간대 추천
- 예: "YouTube는 저녁 8시, Instagram은 점심 12시"

#### 반복 배포
- 매일/매주/매월 자동 발행
- 콘텐츠 시리즈 일괄 예약

#### 대기열 관리
- 발행 대기 중인 콘텐츠 목록
- 우선순위 설정
- 실패 시 재시도 옵션

- 효과 제어 패널

## 5. Technical Requirements

### 5.1 시스템 아키텍처

```
[Client]
  ↓
[Next.js Frontend (Vercel)]
  ↓
[Next.js API Routes]
  ├─→ [Supabase (DB + Auth + Storage)]
  ├─→ [Claude API (Script)]
  ├─→ [PiAPI (Image/Video)]
  ├─→ [ElevenLabs (Voice)]
  ├─→ [Creatomate (Compositing)]
  ├─→ [Mux (Live Streaming)]
  └─→ [YouTube/Facebook/Naver APIs]
```

### 5.2 성능 요구사항

| 항목 | 목표 |
|------|------|
| 페이지 로드 시간 | < 2초 |
| AI 시나리오 생성 | < 30초 |
| 영상 합성 시간 | < 5분 (60초 영상 기준) |
| 동시 사용자 | 1,000명 |
| 업타임 | 99.5% |

### 5.3 보안 요구사항

- HTTPS 필수
- OAuth 2.0 인증
- API 키 암호화 저장
- Rate Limiting (100 req/min per user)
- CSRF 보호

### 5.4 확장성

- Horizontal Scaling (Vercel Edge Functions)
- Supabase Connection Pooling
- CDN 활용 (Cloudflare)
- Queue 시스템 (BullMQ) - 대량 처리

## 6. UI/UX Requirements

### 6.1 디자인 원칙

1. **간결함**: 필요한 정보만 표시
2. **일관성**: 모든 페이지 동일한 패턴
3. **빠른 피드백**: 로딩 상태 명확히 표시
4. **접근성**: WCAG 2.1 AA 준수

### 6.2 주요 화면 구성

#### 대시보드
- 최근 프로젝트 (카드 형태)
- 채널 상태 (연결/미연결)
- 이번 주 통계 (조회수, 좋아요)
- 빠른 액션 버튼 (새 숏폼, 라이브 시작)

#### 숏폼 생성 페이지
- 좌측: 설정 패널 (키워드, 옵션)
- 중앙: 미리보기 화면
- 우측: 진행 상태 (Step Indicator)

#### 채널 관리 페이지
- 연결된 채널 카드 (플랫폼 로고, 이름, 상태)
- "새 채널 연결" 버튼
- 채널별 통계 요약

#### 배포 캘린더
- 월간 뷰 (기본)
- 예약된 콘텐츠 표시 (채널별 색상)
- 드래그 앤 드롭 일정 조정

### 6.3 반응형

- **Desktop**: 1440px 기준 최적화
- **Tablet**: 768px 이상 지원
- **Mobile**: Phase 2에서 지원 (현재는 Desktop 우선)

## 7. Non-Functional Requirements

### 7.1 성능

- **응답 시간**: API 응답 < 1초
- **동시 접속**: 최소 500명
- **영상 처리**: 1분 영상 5분 이내 완성

### 7.2 가용성

- **업타임**: 99.5% (월 3.6시간 다운타임 허용)
- **백업**: 일일 자동 백업
- **복구**: 24시간 이내 복구

### 7.3 확장성

- **사용자**: 10,000명까지 지원
- **프로젝트**: 사용자당 무제한
- **스토리지**: 플랜별 차등 (5GB ~ 1TB)

### 7.4 보안

- **데이터 암호화**: AES-256
- **API 보안**: OAuth 2.0 + JWT
- **침입 탐지**: Cloudflare WAF
